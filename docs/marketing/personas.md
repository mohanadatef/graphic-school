# User Personas

## Overview

This document defines the primary user personas for Graphic School 2.0, helping to understand user needs, goals, and pain points.

## Primary Personas

### 1. Admin (Administrator)

**Demographics:**
- Age: 30-50
- Role: School Administrator, Operations Manager
- Technical Skill: Intermediate to Advanced
- Experience: 5+ years in education administration

**Goals:**
- Manage all courses and groups efficiently
- Track student enrollments and attendance
- Configure system settings
- Customize branding and content
- Generate reports and analytics

**Pain Points:**
- Managing multiple tools for different tasks
- Time-consuming administrative work
- Difficulty tracking student progress
- Lack of customization options
- Complex enrollment processes

**Needs:**
- Complete control over system
- Streamlined workflows
- Comprehensive reporting
- Easy configuration
- Professional website

**Key Features Used:**
- Course and group management
- Enrollment approval
- Attendance overview
- User management
- System settings
- CMS page builder

**Quote:**
*"I need a system that gives me complete control while making my job easier, not harder."*

### 2. Instructor (Teacher/Trainer)

**Demographics:**
- Age: 25-55
- Role: Course Instructor, Trainer
- Technical Skill: Basic to Intermediate
- Experience: 2+ years teaching

**Goals:**
- Manage assigned groups easily
- Take attendance quickly
- View student information
- Schedule sessions
- Track student progress

**Pain Points:**
- Complex attendance systems
- Difficulty accessing student information
- Time-consuming administrative tasks
- Lack of organization tools
- Poor session management

**Needs:**
- Simple group management
- Quick attendance taking
- Easy student access
- Clear session schedule
- Minimal administrative work

**Key Features Used:**
- My groups
- Take attendance
- View students
- Session management
- Calendar view

**Quote:**
*"I want to focus on teaching, not paperwork. The system should make my job easier."*

### 3. Student (Learner)

**Demographics:**
- Age: 18-40
- Role: Student, Course Participant
- Technical Skill: Basic
- Experience: Varies

**Goals:**

- Check session schedule

- Track progress

**Pain Points:**

- Unclear course structure
- Hard to check attendance
- Poor organization
- Confusing interfaces

**Needs:**
- Easy access to information
- Clear course structure
- Simple attendance checking
- Organized interface
- Mobile-friendly

**Key Features Used:**
- My courses
- My group
- My sessions
- Attendance history
- Profile

**Quote:**
*"I just want to see my courses and schedule easily. Keep it simple."*

## Persona Scenarios

### Admin Scenario: Enrollment Rush

**Situation:** 50 new enrollment requests received

**Current Pain:** Manual processing takes hours

**With Graphic School 2.0:**
1. View all pending enrollments
2. Review each quickly
3. Approve/reject with one click
4. Auto-assign to groups
5. Complete in 30 minutes

**Outcome:** 80% time savings, better organization

### Instructor Scenario: Taking Attendance

**Situation:** Need to take attendance for 25-student session

**Current Pain:** Paper-based, time-consuming, error-prone

**With Graphic School 2.0:**
1. Open session attendance
2. Mark each student (present/absent/late)
3. Add notes if needed
4. Save with one click
5. Done in 2 minutes

**Outcome:** 90% time savings, accurate records

### Student Scenario: Checking Schedule

**Situation:** Student wants to check upcoming sessions

**Current Pain:** Need to contact admin or instructor

**With Graphic School 2.0:**
1. Login to dashboard
2. View "My Sessions"
3. See all upcoming sessions
4. Check dates, times, locations
5. Access meeting links

**Outcome:** Self-service, instant access

## User Journey Maps

### Admin Journey

1. **Discovery:** Learn about platform
2. **Evaluation:** See features and benefits
3. **Trial:** Test with sample data
4. **Purchase:** Subscribe to plan
5. **Setup:** Complete setup wizard
6. **Usage:** Daily administration
7. **Growth:** Scale with institution

### Instructor Journey

1. **Onboarding:** Receive account
2. **First Login:** Explore dashboard
3. **Group Assignment:** Get assigned groups
4. **Daily Use:** Manage groups and attendance
5. **Engagement:** Interact with students

### Student Journey

1. **Discovery:** Find course on website
2. **Enrollment:** Submit enrollment request
3. **Approval:** Wait for approval
4. **Access:** Login to dashboard
5. **Learning:** Attend sessions, track progress
